# Insert Bulk Load Data

This pipeline automates the execution of SQL scripts against a PostgreSQL database. It is composed of two stages: a conversion stage that transforms individual INSERT statements into batched transactions, and an execution stage that loads the resulting files into the database in the correct order.

## Requirements

- Python 3.10 or later
- PostgreSQL 16
- `psycopg2-binary`
- `tqdm`

## Virtual Environment

Create and activate a virtual environment before installing dependencies.

**Windows:**
```sh
python -m venv venv
venv\Scripts\activate
```

**macOS / Linux:**
```sh
python -m venv venv
source venv/bin/activate
```

## Installation
```sh
pip install psycopg2-binary tqdm
```

For the general pipeline setup, refer to [PostgreSQL Pipeline Executor](https://github.com/Doc-UP-AlejandroJaimes/sql-101-mastering/tree/main/content/fintech-accelator/scripts/pipelines/insert-data).

## Expected Directory Structure

Before running the pipeline, the `data/sql` directory must have the following structure:
```
data/sql/
├── bulk-load/                          # Generated by convert_to_bulk_insert.py
├── catalogs/
├── customers/
│   ├── customers_20260328.sql
│   └── addresses_20260328.sql
├── payments/
│   ├── orders_20260328.sql
│   └── orders_items_20260328.sql
├── shipments/
│   └── shipment_orders_20260328.sql
├── orders_variety_c20260328.sql
├── orders_items_variety_c20260328.sql
└── shipment_orders_variety_c20260328.sql
```

The three `*_variety_*.sql` files must be located at the root of `data/sql`. These are the source files consumed by the conversion stage. The `bulk-load/` directory is created automatically by the first script.

## Execution

### Stage 1 — Convert to Bulk Insert

Navigate to the pipeline directory and run the conversion script. This reads the three variety files from `data/sql/` and generates batched INSERT files inside `data/sql/bulk-load/`.
```sh
cd scripts/pipelines/insert-bulk-load-data

python convert_to_bulk_insert.py --batch-size 10000
```

Optional arguments:

| Argument | Default | Description |
|---|---|---|
| `--batch-size` | `10000` | Number of rows per `BEGIN/COMMIT` block |
| `--data-path` | `../../../data/sql` | Path to the directory containing the variety SQL files |
| `--output-dir` | `bulk-load` | Name of the output subdirectory inside `--data-path` |

After execution, the `bulk-load/` directory will contain:
```
bulk-load/
├── orders_variety_c20260328_bulk_batches.sql
├── orders_items_variety_c20260328_bulk_batches.sql
└── shipment_orders_variety_c20260328_bulk_batches.sql
```

### Stage 2 — Execute SQL Files

Run the insertion pipeline. Two modes are available depending on whether the standard files have already been loaded.

**Full load** — standard files followed by bulk load files:
```sh
python sql_insert_bulk_load.py \
  --user admin \
  --password "test25**" \
  --db-name customers_db \
  --sql-dir ../../../data/sql \
  --delay 3
```

**Bulk load only** — skips standard files and loads only the batched variety files:
```sh
python sql_insert_bulk_load.py \
  --user admin \
  --password "test25**" \
  --db-name customers_db \
  --sql-dir ../../../data/sql \
  --bulk-load-only \
  --delay 3
```

Available arguments:

| Argument | Default | Required | Description |
|---|---|---|---|
| `--user` | — | Yes | PostgreSQL username |
| `--password` | — | Yes | PostgreSQL password |
| `--db-name` | `customers_db` | No | Target database name |
| `--host` | `localhost` | No | PostgreSQL host |
| `--port` | `5433` | No | PostgreSQL port |
| `--sql-dir` | `.` | No | Root directory containing the SQL subdirectories |
| `--delay` | `1.0` | No | Seconds to wait between file executions |
| `--max-retries` | `3` | No | Maximum connection retry attempts |
| `--bulk-load-only` | `False` | No | When present, only bulk load files are executed |

## Execution Order

The pipeline processes files in the following order when running a full load:
```
1. customers/customers_20260328.sql
2. customers/addresses_20260328.sql
3. payments/orders_20260328.sql
4. payments/orders_items_20260328.sql
5. shipments/shipment_orders_20260328.sql
6. bulk-load/orders_variety_c20260328_bulk_batches.sql
7. bulk-load/orders_items_variety_c20260328_bulk_batches.sql
8. bulk-load/shipment_orders_variety_c20260328_bulk_batches.sql
```

This order is determined by foreign key dependencies. Referencing tables must be populated before the tables that reference them. If any file fails, the pipeline stops immediately and reports the number of successfully executed files.

## Version Control

The `bulk-load/` directory is excluded from version control because its contents are generated artifacts. Ensure the following entry exists in your `.gitignore`:
```
data/sql/bulk-load/
```

The variety source files (`*_variety_*.sql`) are also large generated files and may be excluded depending on the project policy. The files that must be committed are the scripts under `scripts/pipelines/insert-bulk-load-data/` and the standard SQL files under `data/sql/customers/`, `data/sql/payments/`, and `data/sql/shipments/`.

## Performance Considerations

### Indexes on Foreign Key Columns

By default, PostgreSQL creates indexes automatically only for `PRIMARY KEY` and `UNIQUE` constraints. Foreign key columns are not indexed automatically, which can cause significant performance degradation when triggers or queries perform lookups on those columns during bulk load operations.

The `ship.shipment_orders` table has a trigger that validates uniqueness by querying `order_id` on every insert. Without an index on that column, PostgreSQL performs a sequential scan on the entire table for each row, and the cost increases as the table grows within the same load operation.

Ensure the following index exists before running the bulk load:
```sql
CREATE INDEX idx_shipment_orders_order_id 
ON ship.shipment_orders(order_id);
```

The impact is significant. During load testing on this pipeline the same 30-batch file produced the following results:

| Scenario | Time per batch | Total time (30 batches) |
|---|---|---|
| Without index | ~50 seconds | ~25 minutes |
| With index | <1 second | ~14 seconds |
